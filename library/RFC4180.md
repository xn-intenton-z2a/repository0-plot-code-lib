Purpose and scope
- RFC 4180 defines a common format for Comma-Separated Values (CSV) files and their MIME type text/csv.

Record and field delimiting rules
- Records are delimited by CRLF (\r\n). Implementations often accept lone LF as a practical tolerance but canonical format uses CRLF.
- Each record consists of fields separated by commas.

Quoting and escaping rules
- Fields containing commas, double-quotes, or CRLF must be enclosed in double-quotes.
- A double-quote appearing inside a field must be escaped by preceding it with another double-quote. Example: "She said ""Hello""" represents field content: She said "Hello".

Header row and record ordering
- The first record may contain a header line with field names; the RFC does not mandate headers but acknowledges them as common practice.
- All records should have the same number of fields.

Implementation notes for CSV parsing
- Robust parsers must handle quoted fields with embedded newlines and escaped quotes.
- When reading large CSV, stream parsing is recommended to avoid loading entire file into memory.
- For mission: CSV files are expected with columns time,value; implement a parser that splits records correctly and converts the time column to numeric or ISO timestamp as needed.
