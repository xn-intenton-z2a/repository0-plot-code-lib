Overview:
RFC 4180 defines a commonly used CSV format. Key rules for implementation are presented here so a parser can handle typical CSV files, including fields with embedded commas, quotes and newlines.

Record and field rules:
- Each record is located on a separate line, delimited by CRLF (\r\n). Parsers should also accept LF-only for compatibility.
- The last record in the file may or may not have an ending line break.
- There may be an optional header line; headers are a row with column names.

Field delimiter and quoting:
- The comma (,) is the field separator.
- Fields that contain embedded commas, double-quotes, or line breaks must be quoted with double-quotes.
- A double-quote appearing inside a field must be escaped by preceding it with another double-quote ("" becomes a literal ").

Quoted field details:
- A quoted field begins with a DQUOTE and ends with a matching DQUOTE.
- Inside a quoted field any character is allowed including CR and LF; CRLF inside quotes are part of the field content.
- Two consecutive DQUOTE characters inside a quoted field represent a single literal DQUOTE in the field value.

Implementation algorithm (robust, streaming):
1) Read input as a stream of characters; track state: inQuote (boolean), field buffer, record array.
2) For each character c:
   - If c is DQUOTE:
     if inQuote and next char is DQUOTE then append a DQUOTE to buffer and consume next char; else toggle inQuote state.
   - Else if c is COMMA and not inQuote: end current field, push to record.
   - Else if c is CR or LF and not inQuote: end field and end record (handle CRLF together as single terminator).
   - Else append c to current field buffer.
3) At EOF, flush last field/record if present.

Time series specifics (time,value):
- Expect two columns named time and value in the header row or documented externally.
- time: prefer ISO-8601 strings; support parsing via Date.parse or manual epoch parsing.
- value: parse as Number (use parseFloat and Number.isFinite to validate).

SUPPLEMENTARY DETAILS
- Avoid naive line-splitting by '\n' before handling quotes; quoted fields may contain newlines.
- Provide a fast path for simple CSV with no quotes to speed up typical cases.
- For large files parse as streaming (line-by-line/state machine) to avoid high memory use.

REFERENCE DETAILS
- RFC 4180 canonical rules: field separator = ','; quote char = '"'; escaped quote = '""'; CRLF is preferred line terminator but LF-only must be accepted.
- Exact escaping rule: two consecutive double-quotes within a quoted field unescape to one double-quote character.
